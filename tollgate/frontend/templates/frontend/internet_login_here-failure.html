If not, see . {% endcomment %} {% block title %}internet login error: cannot find mac{% endblock %} {% block content %}

There was a problem resolving your MAC address, and it couldn't be found in the ARP cache. Please contact the network administrator for assistance.

This happens primarily for one of these reasons:

  1. You are accessing tollgate from the server itself.

  2. The platform that tollgate is running on (the server) is not supported. Currently only Linux is supported.

    In development, running the software on other platforms will throw this error, and can be ignored. In production it can not.

  3. The server's ARP cache can not be accessed. This could be caused by restrictive SELinux policies.
  4. The server's ARP cache is full. Please see the README for information on how to tune the ARP cache, and performance tuning on larger networks.
